# PHP 基础：

# PHP 语言规则

PHP 脚本可放置在文件的任何位置

PHP 脚本以结尾，语句以分号 (;) 结尾

php 文件通常包含 html 标记及 php 脚本代码，默认扩展名为‘.php’

所有关键字和用户定义的函数名、类名均不区分大小写

变量名区分大小写

# 属于弱类型变量，不需要事先声明，可以通过赋值进行初始化为任何数据类型

# 变量规则：

1、变量以 $ 开头，其后是变量名

2、变量名必须以字母或下划线开头，不能以数字开头

3、变量名只能包含数字、字母、下划线

4、区分大小写，建议小写

5、无创建变量的命令，变量在其首次被赋值时自动创建

6、不必声明变量的数据类型，使用时会自动转换成相应的数据类型

# 会话控制

# 会话的概念

计算机中，指客户端与服务器之间连续发生的一系列请求和响应过程

# 什么是会话控制

http 是一个无状态的协议，无法维护两个事务之间的联系，当一个用户请求一个页面后请求另外一个页面时，http 无法告诉我们这两个请求来自同一个变量，会话控制就是能够在网站中追踪一个变量，可以做到对用户的支持，并根据授权和用户身份显示不同的内容，不同的页面

# http 协议本身无状态

http 协议中，无论是客户端还是服务器都没有必要记录彼此过去的行为，每一次请求之间都是独立的

# 会话控制的实现方法

php 会话通过会话 ID 实现，用户每次登录得到的会话 ID 是唯一的，可以保存在用户的机器里，或者通过 url 在网络上传输。会话过程有 web 服务器端的程序进行开启，一旦开启了一个会话，服务器程序就要为这个会话创建一个独立的存储结构来保存该会话的状态信息，同一个会话中的访问请求都可以且只能访问属于该会话的存储结构中的状态信息

**Cookie 和 Session 就是能够记录用户状态的技术

# Cookie

在 http 协议下，服务器或脚本可以维护客户端信息的一种方式，是 web 服务器保存在用户浏览器上的文件，可以包含有关的用户信息，当访问服务器是，服务器可以访问到 Cookie 的信息

# 设置 Cookie

php 用 seicookie 函数设置 cookie

Cookie 在传输过程中是 http 头的一部分，用于浏览器和服务器之间传递信息

必须在 html 文件本身的内容输出之前调用 setcookie（）函数，调用之前不能有任何内容

# setcookie（）语法

setcookie（name,value,[][][][][expire],[path],[domain],[secure]）

name：设置 cookie 名

value：设定 Cookie 值

expire：设置有效期，单位：秒

path：服务器的有效路径

domain：设定有效域名

secure：设定是否仅在 http 安全连接是才设定

# 接收和处理 Cookie

php 会自动从 web 服务器就收 http 头并且分析他，$_COOKIE 变量用于取回 cookie 的值

# 删除 cookie

1、调用只有 name 参数的 setCookie（）函数，将 value 置为空，即可删除

2、设置 Cookie 的失效时间，应当使过期时间变更为过去的时间点

** 当一个 Cookie 被删除是，它在当前页面一九有效，如果把 cookie 保存为浏览器进程，即浏览器关闭或刷新后就失效了
